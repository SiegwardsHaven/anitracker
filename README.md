## Features

- **Search** anime & manga with advanced filters — genres, score, status, type, and sort order
- **Detail pages** with banner art, metadata sidebar, score badge, genre pills, synopsis, episode grids, trailer embeds, relations, and external links
- **My Lists** — MAL-style table with status tabs, color-coded progress bars, inline search, and column sorting
- **Edit modal** — update status, progress, score, and review notes from your list
- **User accounts** — registration, login, password reset with security questions
- **Profile** — customizable avatar, account management, list wipe/delete with verification
- **Statistics** — scores distribution, status breakdown, genre charts, and top-rated entries
- **Trending carousel** — top-rated anime on the homepage with 8-hour cache
- **Dark theme** — sleek dark UI with pink accent, inspired by MAL

## Tech Stack

| Layer | Technology |
|-------|-----------|
| **Backend** | Python 3.11, Flask 3.x |
| **Database** | MongoDB Atlas (cloud) |
| **Auth** | Flask-Login, bcrypt |
| **Anime data** | [Jikan v4](https://jikan.moe/) (MAL API) |
| **Cover art** | [AniList GraphQL](https://anilist.gitbook.io/anilist-apiv2-docs/) |
| **Production** | Gunicorn on Render |

## Local Development

### Prerequisites
- Python 3.11+
- A [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) free cluster (or local MongoDB)

### Setup

```bash
# Clone the repo
git clone https://github.com/YOUR_USERNAME/anitracker.git
cd anitracker

# Create virtual environment
python -m venv .venv
.venv\Scripts\activate      # Windows
# source .venv/bin/activate  # macOS/Linux

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Edit .env with your MongoDB URI and a secret key
```

### Run

```bash
python app.py
```

Open [http://127.0.0.1:5000](http://127.0.0.1:5000)

## Deploy to Render

1. Push this repo to GitHub
2. Go to [render.com](https://render.com) → **New** → **Web Service**
3. Connect your GitHub repo
4. Render auto-detects `render.yaml` — just add your environment variables:
   - `MONGO_URI` — your MongoDB Atlas connection string
   - `SECRET_KEY` — auto-generated by Render
5. Click **Deploy**

> **Important:** In MongoDB Atlas, add `0.0.0.0/0` to your IP Access List so Render's servers can connect.

## Environment Variables

| Variable | Description |
|----------|-------------|
| `MONGO_URI` | MongoDB Atlas connection string |
| `SECRET_KEY` | Flask session secret (auto-generated on Render) |
